# マーク採点の使い方

Mark2 形式のマークシートを自動で読み取り、採点するモードです。

---

## ワークフロー概要

```
座標ファイル読込 → スキャン画像読込 → OMR認識 → マークチェック → 採点 → 結果出力
```

---

## 1. ファイルの準備

マーク採点に必要なファイルは 3 つです。

| ファイル | 説明 |
|---|---|
| **Mark2 座標ファイル** | Mark2 テンプレートサイトからダウンロードした座標 Excel（`.xlsx`）形式 |
| **正答ファイル** | 各問題の正答を記入した Excel（`.xlsx`）形式 |
| **スキャン画像** | 答案をスキャンした画像（JPEG / PNG / PDF 形式） |

!!! info "Mark2 座標ファイルとは"
    [Mark2（慶應義塾大学 SFC 研究所）](https://github.com/Mark2OSS/Mark2) で作成する、マークシート上のマーク位置を定義した Excel ファイルです。95 × 42 pt の A4 座標系を使用しています。

!!! tip "模範解答の表示方式"
    模範解答の表示方式については、以下のサイトも参考になります。
    
    :material-link: [デジタル採点 All in One — Object Pascalと僕と](https://coding-tips-memoranda.com/%E3%83%87%E3%82%B8%E3%82%BF%E3%83%AB%E6%8E%A1%E7%82%B9-all-in-one/)

## 2. メイン画面の操作

モード選択で **「マーク採点」** を選ぶと、マーク採点専用のメイン画面が開きます。

画面の指示に従って、座標ファイル → 正答ファイル → スキャン画像フォルダの順に読み込んでください。

![マーク採点メイン画面](../images/02a_main_mark_only.png){ .screenshot }
<span class="caption">マーク採点モードのメイン画面</span>

---

## 3. 閾値キャリブレーション

OMR（光学マーク認識）では、マークの塗りつぶし度合いを「黒い面積の比率」で判定します。

採点侍は **K-means クラスタリング** と **大津の二値化法** を組み合わせて、閾値を自動で最適化します。

キャリブレーション画面では、自動推定された閾値を確認し、必要に応じて手動で微調整できます。

![閾値キャリブレーション](../images/17_threshold_calibrator.png){ .screenshot }
<span class="caption">閾値キャリブレーション画面</span>

---

## 4. マークチェック

OMR の認識結果をもとに、以下のエラーを自動検出します。

- **未マーク**: どの選択肢もマークされていない
- **ダブルマーク**: 複数の選択肢がマークされている

エラーがある場合は、マークチェック画面で 1 件ずつ確認し、GUI 上で修正できます。

![マークチェック画面](../images/18_mark_checker.png){ .screenshot }
<span class="caption">マークチェック画面 — エラーの確認と修正</span>

## 5. 描画設定

採点結果を答案画像に描画する際の設定をカスタマイズできます。

○×マークの大きさ、色、位置、得点テキストの表示位置などを調整できます。

![合計点の位置選択](../images/13_select_total_position.png){ .screenshot }
<span class="caption">合計点の表示位置を選択</span>

## 7. 結果の出力

採点が完了すると、以下のファイルが自動生成されます。

| 出力先 | 内容 |
|---|---|
| `01_Results/` | 生徒別成績サマリー Excel |
| `02_Graded_Detail/` | 採点済み答案画像（○×マーク・得点付き） |
| `03_Final_Report/` | 試験統計 Excel、CTT 分析 PDF |
