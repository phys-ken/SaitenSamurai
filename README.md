# 採点侍 (SaitenSamurai)

**テスト採点統合アプリケーション v4.1**

普通紙のマークシート採点、記述式の採点、またはその両方を組み合わせた採点を、
1 つのアプリでワンストップ処理できるデスクトップアプリケーションです。

起動すると **モード選択画面** が表示されます。テストの形式に合わせて選んでください。

<p align="center">
  <img src="docs/images/01_startup_mode_dialog.png" alt="モード選択画面" width="420">
</p>

| モード | 用途 | 必要なもの |
|---|---|---|
| **マーク採点** | マークシートのみの試験 | スキャン画像 + Mark2 座標ファイル |
| **マーク＋記述** | マーク式と記述式が混在する試験 | スキャン画像 + Mark2 座標ファイル |
| **記述採点** | 記述式のみの試験 | スキャン画像のみ（座標ファイル不要） |

> **上流プロジェクト**
>
> 本プロジェクトは、慶應義塾大学 SFC 研究所で開発されたマークシートシステム
> **[Mark2](https://mark2.sfc.keio.ac.jp/ja/)** の座標系 (595×842 pt) および
> OMR 認識ロジックを基盤としています。
>
> 記述式採点の設計は、作者が以前に公開したデジタル採点ソフト
> **[採点斬り 2021](https://phys-ken.github.io/saitenGiri2021/)** の
> キーボード入力方式を参考にしています。

## 機能一覧

### マーク採点関連

| 機能 | 説明 |
|---|---|
| コーナーマーカー検出 | 四隅の黒マーカーを自動検出し射影変換で傾き補正 |
| マーク認識 (OMR) | 二値化 + 面積比率でマーク有無を判定 |
| 閾値キャリブレーション | K-means + Otsu 法で色閾値・面積閾値を自動推定 |
| マーク式採点 | 正答テンプレートとの照合。複数正答・配点・観点に対応 |
| マークチェック | 未マーク・ダブルマーク・不正値を検出し GUI で修正 |

### 記述採点関連

| 機能 | 説明 |
|---|---|
| 記述問題の領域設定 | 答案画像上でマウスで記述欄の位置・配点・観点を設定 |
| 記述式採点（1枚ずつ） | 1 人ずつ切り出し画像を表示し、キーボードで得点入力（自動で次の生徒へ進む） |
| 記述式採点（一覧） | 全生徒の切り出し画像をグリッド表示し、得点ボタン＋クリックで一括採点 |
| 〇正解 / ×不正解ボタン | 〇ボタン（m キー）で満点、×ボタン（b キー）で 0 点をワンタッチ付与 |
| 背景色フィードバック | 採点直後にキャンバス背景が変化（満点=薄青 / 0点=薄赤 / 中間=薄橙） |
| 未採点フィルタ | 「未採点のみ表示」チェックで既採点者をスキップ |
| 完了ダイアログ | 全員の採点完了時に自動通知（問題選択に戻る / 採点を続ける） |
| 答案プレビュー | 一覧モードで「📷 答案を表示」ボタンから元画像を確認 |
| 記述採点の確認 | 採点結果を一覧で確認・修正 |

### 出力・集計・分析

| 機能 | 説明 |
|---|---|
| 採点済み答案画像 | ○×△マーク・得点・観点番号を答案上に描画 |
| 描画詳細設定 | 選択中のモードに応じた設定項目のみ表示（マーク/記述/両方） |
| 合計点描画 | 指定位置に合計得点を描画（マーク式 + 記述式統合） |
| 氏名トリミング | 氏名エリアを一括切り出しして集計シートに埋め込み |
| 生徒別サマリー Excel | 生徒ごとの設問別得点・観点別得点の一覧表 |
| 試験統計 Excel | 得点分布・設問別正答率・観点別統計のグラフ付きシート |
| CTT 分析 | 古典テスト理論に基づく項目分析 (α 係数, P 値, D 値, I-T 相関) |
| CTT レポート PDF | matplotlib + ReportLab によるグラフ付き PDF レポート |
| R 連携エクスポート | exametrika パッケージ用の分析キット (CSV + R スクリプト + Rmd) |

### 共通機能

| 機能 | 説明 |
|---|---|
| PDF 入出力 | PDF → 画像変換 (入力)、採点済み画像 → 統合 PDF (出力) |
| セッション保存・復元 | 作業状態を JSON で保存し次回起動時に復元 |
| 処理中断 | 長時間処理中に「中断」ボタンで安全にキャンセル |

| exe 配布 | PyInstaller による単一ファイル exe ビルド |

## 解答用紙の準備

採点モードによって、解答用紙の要件が異なります。

### マーク採点・マーク＋記述モードの場合

**Mark2 テンプレートを使った専用の解答用紙** が必要です。

#### 1. テンプレートを入手する

Mark2 公式サイトからテンプレートをダウンロードしてください。

> **テンプレート配布ページ**: https://mark2.sfc.keio.ac.jp/ja/templates

「学力テスト」カテゴリによく使われるテンプレートがあります。

| テンプレート名 | 内容 |
|---|---|
| M2-03-001 | 4 択 × 100 問（国語・英語向け） |
| M2-03-002 | 10 択 × 50 問（数学向け・0〜9 の数字解答） |
| M2-03-004 | 4 択 × 90 問（国語・英語向け・番号記入式） |
| M2-03-005 | 10 択 × 45 問（数学向け・番号記入式） |

zip を展開すると以下の 3 つが入っています:

- **印刷原稿（PDF）** — そのまま印刷して使います
- **印刷原稿（PPTX）** — PowerPoint で問題文を追記したい場合に
- **座標ファイル（Excel）** — 採点侍に読み込ませるファイルです（⚠ 編集しないでください）

#### 2. 解答用紙のポイント

- **用紙**: **普通紙（コピー用紙）で OK** です。特殊な用紙は不要
- **四隅のマーカー**: テンプレートには四隅に **黒い四角形のコーナーマーカー** が印刷されています。これが自動傾き補正 + 座標合わせの基準になるため、**スキャン時に四隅すべてが画像内に映っている** 必要があります
- **筆記具**: 鉛筆またはシャープペンシル推奨（ボールペンでも可だが認識精度が下がる場合あり）

#### 3. スキャンのコツ

| 項目 | 推奨 |
|---|---|
| 保存形式 | JPEG または PNG |
| 解像度 | **300 DPI** 推奨（200 DPI でも可） |
| カラー | グレースケールでもカラーでも OK |
| 枚数 | **1 枚の画像 = 1 人分の解答用紙** |
| 傾き | 多少の傾きは自動補正されますが、大きく傾いていると失敗します |

> 💡 PDF でスキャンした場合もそのまま読み込めます（採点侍が自動で画像に変換）。

スキャンした画像は **1 つのフォルダにまとめて** 保存してください。

---

### 記述採点モードの場合

**解答用紙の形式に決まりはありません。** 四隅のコーナーマーカーも不要です。

手書きの答案用紙、白紙のレポート、学校の定期試験の解答用紙など、
どんな形式の用紙でもスキャンすれば採点できます。

#### 準備するもの

- **答案画像**: JPEG / PNG（1 ファイル = 1 人分の答案）
- 座標ファイルは **不要**

#### ファイル名が生徒名になります

記述採点モードでは、画像ファイル名がそのまま集計シートの「生徒ID」として使われます。
スキャン後、**誰の答案かわかるファイル名** にリネームしてください。

```
例:
  01_山田太郎.jpg
  02_鈴木花子.jpg
  03_田中一郎.jpg
```

#### スキャンの注意

- **上下の向き**: 自動回転はされないため、正しい向きでスキャンしてください
- **解像度**: 記述欄の文字が読める程度であれば OK（200〜300 DPI 程度）
- **形式**: JPEG / PNG。PDF の場合はアプリ内で画像に変換されます

## 採点モード別の流れ

GUI は 3 ステップ構成です。モードによって UI と操作が変わります。

### マーク採点モード

<p align="center">
  <img src="docs/images/02a_main_mark_only.png" alt="マーク採点モード画面" width="700">
</p>

```
Step 1: 画像＋座標ファイル選択 → OMR 認識実行
Step 2: マークチェック → 合計点位置設定 → 採点済み答案生成
Step 3: 集計（サマリー Excel・CTT 分析・PDF）
```

### マーク＋記述モード

<p align="center">
  <img src="docs/images/02b_main_mark_descriptive.png" alt="マーク＋記述モード画面" width="700">
</p>

```
Step 1: 画像＋座標ファイル選択 → OMR 認識実行 → 記述問題設定
Step 2: マークチェック → 記述採点（1枚ずつ / 一覧を切替可能）→ 採点済み答案生成
Step 3: 集計（マーク＋記述を統合して分析）
```

### 記述採点モード

<p align="center">
  <img src="docs/images/02c_main_descriptive_only.png" alt="記述採点モード画面" width="700">
</p>

```
Step 1: 画像フォルダ選択 → 画像準備 → 記述問題設定
Step 2: 記述採点 → 合計点位置設定 → 採点済み答案生成
Step 3: 集計（記述得点のみで分析）
```

> 記述採点モードでは座標ファイル・OMR 認識・マークチェックは不要です。
> 生徒の識別にはスキャン画像のファイル名がそのまま使われます。

#### 記述採点の 2 つの表示モード

記述採点では、**問題一覧画面** で採点モード（「1枚ずつ」/「一覧（グリッド）」）を選んでから「採点」ボタンを押します。
前回選んだモードは自動で記憶されるため、毎回選び直す必要はありません。
採点中にモードを変更したい場合は、一度「採点完了」で問題一覧画面に戻り、モードを切り替えてから再度「採点」を押してください。

| 表示モード | 操作方法 | 向いている場面 |
|---|---|---|
| **1枚ずつ** | 1 人ずつ大きく表示。キーボード (0–9) で得点入力すると自動で次の生徒へ | 記述内容をじっくり確認したいとき |
| **一覧** | 全生徒をサムネイルでグリッド表示。得点ボタンを選んでからサムネイルをクリック | 同じ得点の答案をまとめて処理したいとき |

一覧モードでは画面上部の **🔍 サイズスライダー** でサムネイルの大きさを自由に変更できます（80px〜800px）。

<p align="center">
  <img src="docs/images/10_descriptive_scorer_list.png" alt="問題選択リスト" width="480">
</p>
<p align="center">
  <img src="docs/images/06_single_question_scorer.png" alt="1枚ずつ採点" width="480">
  <img src="docs/images/20_grid_scoring_mode.png" alt="一覧グリッド採点" width="480">
</p>

#### 記述採点の便利機能

| 機能 | 操作 | 説明 |
|---|---|---|
| **〇 正解ボタン** | クリック or `m` キー | 満点を即座に付与して次の生徒へ進む |
| **× 不正解ボタン** | クリック or `b` キー | 0 点を付与して次の生徒へ進む |
| **背景色フィードバック** | 自動 | 採点後に背景色が一瞬変化（満点=薄青 / 0点=薄赤 / 中間=薄橙） |
| **未採点フィルタ** | チェックボックス | 「未採点のみ」を ON にすると、既に採点済みの生徒をスキップ |
| **未採点ジャンプ** | `Tab` キー | 次の未採点の生徒に直接ジャンプ |
| **完了ダイアログ** | 自動 | 全員の採点完了時に自動で通知ダイアログを表示 |
| **📷 答案を表示** | ボタン（一覧モード） | サムネイルクリックで元画像をOSビューアで確認 |

> 💡 〇/× ボタンは配点が 10 点以上の Entry モードでも使えます。
> `m` キーで満点、`b` キーで 0 点をワンタッチで入力できるため、
> 正答/不正答が明確な問題では大幅な時間短縮になります。

## 出力フォルダ構造

処理結果は画像フォルダ直下に自動生成されます。

```
_saiten_grading_results/
├── 00_Processing/           # 射影変換済み画像（中間ファイル）
├── 01_Results/              # OMR データ・座標・正答定義
│   ├── coordinates.csv
│   ├── answer_key.xlsx
│   └── reading_results/     # 生徒ごとの OMR 結果 Excel
│       └── backup/
├── 02_Graded_Detail/        # 個別採点済み画像
└── 03_Final_Report/         # 最終レポート
    ├── 001_student_summary.xlsx
    ├── 002_exam_summary.xlsx
    ├── 003_CTT_analysis_report.xlsx
    ├── 004_CTT_analysis_report.pdf
    ├── 005_scored_all.pdf
    └── 006_R_analysis_kit/
        ├── scored_data.csv
        ├── item_info.csv
        ├── run_exametrika.R
        └── report_template.Rmd
```

> 記述採点モードでは `00_Processing/` と `01_Results/` の内容は簡略化されます。

## インストール

### 1. リポジトリのクローン

```bash
git clone https://github.com/phys-ken/SaitenSamurai.git
cd SaitenSamurai
```

### 2. パッケージのインストール

```bash
pip install -r requirements.txt
```

| パッケージ | 用途 | 必須/任意 |
|---|---|---|
| opencv-python-headless | 画像処理・OMR 認識 | 必須 |
| numpy | 数値計算 | 必須 |
| pandas | データフレーム操作 | 必須 |
| Pillow | 画像処理・描画 | 必須 |
| openpyxl | Excel 読み書き | 必須 |
| PyMuPDF | PDF → 画像変換 | 任意 |
| matplotlib | CTT レポートのグラフ | 任意 |
| reportlab | CTT レポートの PDF 生成 | 任意 |

## exe 版の利用

ビルド済みの `SaitenSamurai.exe` が配布されている場合は、インストール不要で利用できます。

- 任意の場所に配置してダブルクリックで起動（Python 不要）
- 初回起動時に Windows Defender SmartScreen の警告が出る場合は「詳細情報」→「実行」
- エラー発生時は exe と同じフォルダに `saitensamurai_crash.log` が生成されます

### exe のビルド方法

```bash
build_exe.bat
```

出力先: `dist/SaitenSamurai.exe`

## 起動方法

### Python で実行

```bash
python main_src/saitensamurai.py
```

起動すると **モード選択ダイアログ** が表示されます。
「マーク採点のみ」「マーク＋記述」「記述採点のみ」の中から選んでください。

過去のセッションを再開する場合は「📂 採点再開（セッション復元）」を選択します。

## 注意事項

- **OMR 閾値の調整**: マーク読み取り精度が低い場合は「🔧 Auto」ボタンで自動キャリブレーション。手動スライダーでの微調整も可能
- **マークチェック**: 認識結果の信頼性を確保するため、採点前にマークチェックの実施を推奨
- **Excel ファイルのロック**: 集計実行時に出力先の Excel が開いていないことを確認してください
- **PDF 入力**: PyMuPDF がインストールされていない場合は PDF 選択が使用できません
- **出力画像の解像度上限**: A4 200 DPI 相当（`OUTPUT_SCALE_MAX = 2.0`）

## モジュール構成

`main_src/` 以下の 16 モジュール:

| モジュール | 役割 |
|---|---|
