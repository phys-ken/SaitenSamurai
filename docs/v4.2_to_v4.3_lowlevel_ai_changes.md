## 変更詳細

### 1. EXE 実行時の不具合修正 (`e3d0514`)

#### 意図
PyInstaller で `console=False` でビルドした EXE を起動すると、`sys.stdout` が `None` になり `AttributeError` でクラッシュする問題を修正。また、`multiprocessing` 使用時に `freeze_support()` がないと、EXE 起動時にプロセスが多重起動する問題も修正。

#### 変更ファイル

**constants.py** — `setup_logging()` 関数:
- `logging.StreamHandler(sys.stdout)` → stdout が None のとき `os.devnull` にフォールバック
- `safe_print()` — `sys.stdout is None` のとき早期 return

**saitensamurai.py** — エントリポイント:
- `import multiprocessing` 追加
- `multiprocessing.freeze_support()` を `if __name__ == '__main__':` 直後に追加
- バージョン文字列を `4.1` → `4.3.1` に変更（段階的に変更された）

**main_gui.py** — `_run_box_drawer_thread()`:
- `redirect_stdout / redirect_stderr` で `process_folder()` の出力をキャプチャ・パースする処理を**全削除**
- `process_folder()` を直接呼び出すように簡素化
- **注意**: stdout リダイレクトの削除は、EXE 環境で stdout が None の場合にクラッシュするのを回避する意図だったと思われるが、副作用として TQDM 進捗ログの GUI 表示が失われた可能性がある

**.gitignore**:
- `tests/gui_captures/`, `tests/descriptive_scoring_report/`, `tests/grid_scoring_mode*`, `tests/tmp_safety_check/` を追加

**テスト出力ファイル削除**:
- `tests/gui_captures/` 配下の全 PNG（20ファイル）と `index.html`
- `tests/descriptive_scoring_report/` 配下の `index.html` と PNG 8ファイル

- `messagebox.askyesno()` で「フォルダを開きますか？」と確認
- 「はい」の場合、`subprocess.Popen(f'explorer ...')` でフォルダを開く
